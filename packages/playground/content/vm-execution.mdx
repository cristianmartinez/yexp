# VM Execution Visualizer

An interactive visualization showing how Jext's stack-based virtual machine executes bytecode instructions in real-time.

## How It Works

The Jext VM is a **stack-based virtual machine** that:
1. Reads bytecode instructions sequentially
2. Uses an **instruction pointer (IP)** to track position
3. Manages a **stack** for intermediate values
4. Supports **jumps** for control flow (conditionals, loops)

## Stack Operations

The stack is a **LIFO (Last In, First Out)** data structure:

- **Push**: Add value to top of stack
- **Pop**: Remove and return top value
- **Binary ops**: Pop two values, push result
- **Unary ops**: Pop one value, push result

## How to Use

1. **Enter Expression** - Type any Jext expression in the editor
2. **Set Context** - Switch to the Context tab to add data/state/env
3. **Watch Execution** - The bytecode and stack update in real-time
4. **Control Playback:**
   - Click **Play** to auto-step through execution
   - Use **Step Forward/Backward** to manually control
   - **Reset** to start over

## Understanding the Visualization

**Left Panel - Bytecode Instructions:**
- Blue highlight shows current instruction
- Dimmed text shows already-executed instructions
- Jump targets show where control flow goes

**Right Panel - Stack:**
- Shows current stack state (top at bottom)
- Values animate in/out as they're pushed/popped
- Final result appears when execution completes

This is a **stack-based VM** - all operations work through the stack. No registers, just push and pop!
