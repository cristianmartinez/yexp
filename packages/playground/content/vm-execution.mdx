# VM Execution Visualizer

An interactive visualization showing how Jext's stack-based virtual machine executes bytecode instructions in real-time.

## How It Works

The Jext VM is a **stack-based virtual machine** that:
1. **Reads bytecode** instructions sequentially
2. **Instruction Pointer (IP)** tracks current position
3. **Stack** holds intermediate values (LIFO - Last In, First Out)
4. **Jumps** enable control flow (conditionals, loops, lambdas)

## What to Watch For

### Jump Instructions
Watch the IP jump around when:
- **Conditionals** (`? :`) - Jump based on condition
- **Logical operators** (`&&`, `||`) - Short-circuit evaluation
- **Lambdas** - Jump into function, execute, jump back

### Stack Operations
See how the stack changes:
- **Binary operations** - Pop 2 values, push 1 result
- **Array/object construction** - Pop multiple values, push collection
- **Function calls** - Pop arguments, push return value
- **Spread operators** - Flatten and merge collections

### Complex Patterns
- **Pipelines** - Chain operations left-to-right
- **Template literals** - Evaluate expressions within strings
- **Optional chaining** - Safe navigation with null checks

## Controls

**Playback:**
- **Play/Pause** ▶️ - Auto-step through execution (500ms per step)
- **Step Forward** ⏭️ - Execute next instruction
- **Step Backward** ⏮️ - Go back one step
- **Reset** 🔄 - Start from beginning

**Customization:**
- Switch to **Expression** tab to modify code
- Switch to **Context** tab to change data/state/env
- Changes instantly compile and restart execution

## Understanding the Panels

### Left: Bytecode Instructions
- **Blue highlight** = current instruction executing
- **Gray text** = already executed
- **Format**: `[index] OPCODE arguments`
- **Jump targets** show `-> index` for control flow

### Right: Stack
- **Bottom-up display** (top of stack at bottom)
- **Animations** when values pushed/popped
- **Index labels** `[0]`, `[1]`, etc. from bottom
- **Result** appears when execution completes
