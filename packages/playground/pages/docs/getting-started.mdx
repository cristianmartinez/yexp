# Getting Started

Get up and running with Jext in minutes.

## Installation

Install Jext via npm or bun:

```bash
npm install @jext/core
# or
bun add @jext/core
```

## Basic Usage

```typescript
import { run } from '@jext/core'

// Create an execution context
const context = {
  state: {},
  data: {
    products: [
      { name: 'Laptop', price: 999 },
      { name: 'Mouse', price: 25 },
      { name: 'Keyboard', price: 75 }
    ]
  },
  env: {}
}

// Evaluate an expression
const result = run('data.products.filter(p => p.price < 100)', context)
console.log(result)
// Output: [{ name: 'Mouse', price: 25 }, { name: 'Keyboard', price: 75 }]
```

## Execution Context

Every expression runs with three namespaces:

- **`state`** - Mutable application state
- **`data`** - Immutable input data
- **`env`** - Environment variables/configuration

```typescript
const context = {
  state: { count: 0, user: { name: 'Alice' } },
  data: { items: [1, 2, 3] },
  env: { API_URL: 'https://api.example.com' }
}
```

## Compilation & Execution

For better performance, compile expressions once and reuse:

```typescript
import { compileExpr, evaluate } from '@jext/core'

// Compile to bytecode
const program = compileExpr('data.items |> filter(x => x > 2)')

// Execute multiple times (fast!)
const result1 = evaluate(program, context1)
const result2 = evaluate(program, context2)
```
